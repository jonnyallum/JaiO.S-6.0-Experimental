 Node Contract
 ─────────────
 Input keys  : code (str), language (str), task (str — what the code should do)
 Output keys : analysis (str), corrected_code (str), issues (str)
 Side effects: Supabase PRE/POST checkpoints, CallMetrics telemetry

 Sandboxed code execution and validation agent

 Persona: identity injected at runtime via person

## State Contract
- **State class**: `CodeExecutorState`
- **Input keys**: `task` (str), `context` (str), `thread_id` (str)
- **Output keys**: `output` (str), `error` (str)

## Usage
```python
from agents.code_executor import code_executor_node, build_graph

# Direct node call
result = code_executor_node({"task": "your task here", "context": "", "thread_id": "t1"})

# Via compiled graph
graph = build_graph()
result = graph.invoke({"task": "your task here", "context": "", "thread_id": "t1"})
```

## Error Handling
- `PERMANENT: ...` — input validation failure, do not retry
- `TRANSIENT: ...` — API error, safe to retry (auto-retried 3x)
- `UNEXPECTED: ...` — unknown error, investigate

## Persona
Identity injected at runtime via `personas/config.py`. Override with env vars:
```
PERSONA_CODE_EXECUTOR_NAME="Custom Name"
PERSONA_CODE_EXECUTOR_NICKNAME="Custom Nick"
```
